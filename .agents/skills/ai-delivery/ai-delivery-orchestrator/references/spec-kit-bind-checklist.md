# Spec Kit Bind Checklist

Use this checklist after an official `speckit-specify`, `speckit-plan`, or `speckit-tasks` run and before advancing `.ai-delivery` status.

## Minimal Checklist

- [ ] Read `spec-kit-input.md`, target slice `traceability.json`, and target slice `status.json` together before writing anything.
- [ ] Confirm `requirement_id`, `subreq_id`, and `slice_id` in `spec-kit-input.md` match the target slice package.
- [ ] Confirm every output path being bound was actually generated by the official Spec Kit step.
- [ ] Update only `traceability.json.spec_kit_refs`.
- [ ] Preserve existing `requirement_refs`, `api_refs`, `figma_refs`, `ui_acceptance_refs`, and `interaction_refs` unchanged.
- [ ] Preserve previously bound `spec_paths`, `plan_paths`, or `tasks_paths` when the current bind is partial.
- [ ] Write the generated spec path into `traceability.json.spec_kit_refs.spec_paths` when binding spec output.
- [ ] Write the generated plan path into `traceability.json.spec_kit_refs.plan_paths` when binding plan output.
- [ ] Write the generated tasks path into `traceability.json.spec_kit_refs.tasks_paths` when binding tasks output.
- [ ] Write `spec-kit-binding.json` with `spec_kit_input_path`, `bound_outputs`, `bound_statuses`, `traceability_synced`, `official_upstream_unchanged`, `updated_at`, and `updated_by`.
- [ ] Advance `status.json` only when the corresponding generated output exists and `traceability.json.spec_kit_refs` has been synced.
- [ ] Use `spec_ready`, `plan_ready`, and `tasks_ready` only for outputs that were actually bound in this pass.
- [ ] If slice identity, generated paths, or governed upstream truth conflict, stop and open a blocker instead of binding.

## Result Shape

The minimum `traceability.json` patch should look like:

```json
{
  "spec_kit_refs": {
    "spec_paths": ["specs/.../spec.md"],
    "plan_paths": ["specs/.../plan.md"],
    "tasks_paths": ["specs/.../tasks.md"]
  }
}
```
