# 日志检查清单

当独立的管理支持界面可用时，按以下顺序操作。

1. 确定目标项目和子需求范围
2. 为当前的 API 契约映射操作添加开始日志
3. 在变更产物之前，读取当前的阻塞器、状态和 `traceability.json`
4. 在 `.ai-delivery/` 内执行产物写入
5. 通过治理支持界面记录任何状态转换、阻塞器创建、阻塞器恢复或下游重新验证信号
6. 添加足够详细以支持中断后恢复的完成或失败日志

如果管理支持界面尚不可用：

- 将产物实情保留在 `.ai-delivery/` 中
- 在 `decisions.md` 或 `api-contract-mapping.md` 中记录缺失的治理依赖项
- 不得在认可的架构之外创建第二套状态或日志存储
