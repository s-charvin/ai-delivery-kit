# 双重实情规则

## 实情来源

- 需求拥有功能实情。
- API 契约拥有面向客户端的接口实情。
- Figma 拥有视觉实情。
- `.ai-delivery/` 是业务项目内唯一的工作流产物实情。
- `ai-delivery-admin` 是一个治理支持界面，而非替代性的实情存储。

## 冲突处理

当需求与 API 契约不一致时：

- 停止正常执行
- 在相关需求文件夹中记录不匹配情况
- 将受影响的子需求移至最窄的合适阻塞状态
- 不得编造缺失的字段、端点、错误语义或状态值

## 允许的推断范围

仅允许有限的推断：

- 不改变客户端可见含义的路径或操作分组
- 协议文件格式规范化，如从 YAML 到 JSON 的读取
- 重用已建立的治理命名模式，且不改变需求或 API 实情

所有有限的推断必须显式记录在 `decisions.md` 或 `api-contract-mapping.md` 中。
