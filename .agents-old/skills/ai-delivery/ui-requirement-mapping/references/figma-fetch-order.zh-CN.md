# Figma 获取顺序

1. 识别设计来源和首选提供商顺序
2. 获取目标来源的结构化文件或页面上下文
3. 使用结构化节点负载缩小到候选可执行节点
4. 仅在可执行节点集稳定后才获取辅助结构化产物，如评论、token 或资产
5. 仅在用户要求或有助于人工审阅时获取可选预览或截图

规则：

- 优先使用 `.ai-delivery/figma-cache/` 中的缓存证据
- 使用最小包装器缓存每个产物，保留兼容性元数据以及 `provider`、`artifact_type` 和提供商原生 `raw_payload`
- 保留当前管理员读取器所需的兼容性元数据，如 `figma_file_id`、`node_id`、`last_updated_at` 和 `freshness`
- 保持提供商负载为原始响应格式，而非将其扁平化为自定义缓存结构
- 未为每个声称的可执行节点提供可信的结构化节点负载，不得标记映射完成
- 不得仅使用节点名称、截图、预览或文字描述映射需求点
- 不得将顶层 `SECTION` 用作最终可执行目标
- 仅将预览用作可选支持上下文；它们从不覆盖结构化证据
