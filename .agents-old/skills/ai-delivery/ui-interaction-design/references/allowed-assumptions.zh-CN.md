# 允许的假设

允许作为 `assumed_micro_interaction`：

- 按钮加载呈现
- 悬停、活动和焦点行为
- 提示消息与内联错误优先级
- 空列表基线反馈
- 验证触发时序
- 禁用的控件无法激活
- 请求在途时的提交去重
- 基线键盘可达性
- 有边界的可访问性改进，如更强的焦点可见性或更清晰的状态公告语言
- 减少动效的回退行为
- 状态变化的保守动效时序
- 保持布局的加载呈现，如内联进度或骨架屏
- 业务含义不变时的非阻塞成功反馈

记录规则：

- 将行标注为 `Assumption: Micro Interaction`
- 标识支持性的源差距或 `Source: Existing Pattern`
- 解释为什么该假设保持在业务含义阈值以下
- 解释为什么所选的加载、反馈或动效模式是最轻量且安全的选择
- 如果假设会改变业务语义，则停止并升级

不允许：

- 新的业务分支
- 新的步骤、字段、模态框或确认流程
- 新的权限规则
- 新的页面转场
- 没有需求或 Figma 支持的新异常语义
