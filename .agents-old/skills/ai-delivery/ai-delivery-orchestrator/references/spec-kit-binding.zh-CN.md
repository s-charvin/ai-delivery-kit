# Spec Kit 绑定

- 目的：审核官方 Spec Kit 输出并将其绑定回 `.ai-delivery`，无需重写官方 `spec.md`、`plan.md` 或 `tasks.md`。
- 在每次官方 Spec Kit 步骤之后，对以下内容运行本地绑定传递：
  - `spec-kit-input.md`
  - 生成的 `spec.md`、`plan.md` 或 `tasks.md`
  - `traceability.json`
  - `status.json`
- 绑定传递写入：
  - `spec-kit-binding.json`
- 绑定传递还更新：
  - `traceability.json.spec_kit_refs`
  - 切片 `status.json` 为 `spec_ready`、`plan_ready` 或 `tasks_ready`
- `spec-kit-binding.json` 应记录：
  - 输入包路径
  - 绑定的输出路径
  - 推进了哪个就绪状态
  - 可追溯性同步是否成功
  - 审核元数据，如时间戳和执行者
- 在写入 `traceability.json.spec_kit_refs` 之前，使用绑定检查清单：
  - [Spec Kit 绑定检查清单](spec-kit-bind-checklist.md)
- 如果官方输出与受管控的上游真实数据冲突，不要绑定它。打开一个阻塞器或从 `prepare-speckit-context` 重新运行。
- 反向可追溯性属于 `.ai-delivery` 旁路文件，而非修补后的官方 Spec Kit 指令。
- 示例产物：[contacts-friends-idle/spec-kit-binding.json](examples/contacts-friends-idle/spec-kit-binding.json)
